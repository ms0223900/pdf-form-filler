### US-001：資料庫與型別定義

**作為** 開發者
**我想要** 建立共用型別定義與 Dexie 資料庫 schema
**以便** 後續所有模組有統一的型別基礎與 IndexedDB 儲存層

**輸入格式**：
- spec.md 第 4 節的資料庫設計（`pdfs` 表、`materials` 表）
- spec.md 第 4.3 節的 TypeScript 型別定義（`PersonalInfo`、`SignatureData`、`PDFField`、`PDFDocument`）

**輸出格式**：
- `src/lib/types.ts`：匯出所有共用型別（`PersonalInfo`、`SignatureData`、`PDFField`、`PDFDocument`）
- `src/lib/db.ts`：匯出已初始化的 `db` 實例，含 `pdfs` 與 `materials` 兩個 table

**驗收條件**：
- [ ] `src/lib/types.ts` 存在，內容與 spec.md 4.3 節一致（含 `PersonalInfo`、`SignatureData`、`PDFField`、`PDFDocument`）
- [ ] `src/lib/db.ts` 存在，使用 Dexie 定義 `pdfs` 表（`id`、`name`、`fileData`、`uploadedAt`、`type`）與 `materials` 表（`id`、`name`、`type`、`data`、`createdAt`）
- [ ] `npm run build` 通過，無 TypeScript error
- [ ] `npx tsc --noEmit` 無錯誤

**依賴關係**：
- 無（此為基礎設施層）

**優先級**：P0
**相關功能**：資料庫設計（spec §4）
