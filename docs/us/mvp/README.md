# MVP — 最小可行產品

## 範圍說明

MVP 涵蓋完整的 PDF 填表閉環：**上傳 → 預覽 → 填寫 → 匯出下載**。不包含素材庫、簽名、素材管理等加分功能。

### MVP 第 1 階段（已完成）

基礎流程：上傳 PDF → 預覽 → 自動偵測 AcroForm 欄位 → 填寫 → 匯出。

| 編號 | 標題 | 優先級 | 依賴 | 狀態 |
|------|------|--------|------|------|
| US-001 | 資料庫與型別定義 | P0 | - | ✅ 已完成 |
| US-002 | 封裝 PDF 操作工具函式 | P0 | US-001 | ✅ 已完成 |
| US-004 | 實作儀表板與 PDF 上傳 | P0 | US-001 | ✅ 已完成 |
| US-005a | 實作 PDF 預覽元件 | P0 | US-001 | ✅ 已完成 |
| US-005b | 實作表單欄位面板 | P0 | US-002, US-005a | ✅ 已完成 |
| US-007 | 實作 PDF 匯出功能 | P1 | US-002, US-005a, US-005b | ✅ 已完成 |

### MVP 第 2 階段（新增）

自訂區塊功能：使用者在 PDF 任意位置新增文字或圖片區塊，補足 AcroForm 欄位無法涵蓋的填寫需求。

| 編號 | 標題 | 優先級 | 依賴 | 狀態 |
|------|------|--------|------|------|
| US-010 | 擴充型別與匯出函式支援自訂區塊 | P1 | US-002, US-007 | 📋 待開發 |
| US-011 | 實作自訂文字區塊 UI | P1 | US-005a, US-010 | 📋 待開發 |
| US-012 | 實作自訂圖片區塊 UI | P1 | US-010, US-011 | 📋 待開發 |

## 依賴關係

```
Wave 1（已完成）
US-001 (型別+DB)
  ├──▶ US-002 (pdfUtils)
  │     └──▶ US-005b (欄位面板) ──▶ US-007 (匯出)
  ├──▶ US-004 (儀表板)
  └──▶ US-005a (PDF預覽)
        └──▶ US-005b

Wave 2（新增加）
US-002 ──▶ US-010 (擴充型別+匯出)
              │
              ├──▶ US-011 (文字區塊 UI) ───┐
              │                             ├──▶ US-007 (匯出)
              └──▶ US-011 ──▶ US-012 (圖片區塊 UI) ─┘
US-005a ──▶ US-011
```

## 依賴說明

- **US-010** 依賴 US-002（pdfUtils）和 US-007（匯出流程），因為需要擴充 `fillAndExport` 來處理自訂區塊
- **US-011** 依賴 US-005a（PDF 預覽框架）和 US-010（型別定義），因為需要在預覽畫面上疊加 overlay 且需要自訂區塊型別
- **US-012** 依賴 US-010（型別+匯出）和 US-011（共用 overlay/拖曳基礎架構），因為圖片區塊沿用文字區塊建立的基礎設施

## 開發順序

```
Wave 1（已完成）: US-001 → US-002 + US-004（平行）→ US-005a → US-005b → US-007
Wave 2（新增加）: US-010 → US-011 → US-012
```

- US-010 為基礎建設，必須最先完成
- US-011 與 US-012 因共用 overlay 架構，建議循序開發（US-011 → US-012）
- US-007 不需修改匯出邏輯，但需更新 props 傳遞 customBlocks

## 邊界（禁止事項）

MVP 階段**不應**實作以下功能：
- ❌ 素材庫（materialStore）
- ❌ 素材套用面板
- ❌ 簽名繪製工具
- ❌ 素材管理頁面
- ❌ 預設範本
- ❌ 任何超出上傳→填寫→匯出流程的功能
- ✅ **允許**自訂文字區塊與自訂圖片區塊（屬核心填寫流程延伸）
