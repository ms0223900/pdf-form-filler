### US-010：擴充型別與匯出函式支援自訂區塊

**作為** 使用者
**我想要** 系統支援自訂文字區塊與圖片區塊的型別定義及匯出能力
**以便** 後續 US 能實作 UI 操作並正確將區塊內容寫入 PDF

**輸入格式**：
- 現有 `lib/types.ts` 中的 `PDFField`、`PDFDocument` 型別
- 現有 `lib/pdfUtils.ts` 中的 `fillAndExport` 函式
- 各區塊的定位資料：目標頁碼（0-based）、x/y 座標、寬度、高度

**輸出格式**：
- `lib/types.ts` 新增 `CustomBlock` 聯合型別（`text` | `image`），以及 `CustomTextBlock`、`CustomImageBlock` 介面
- `lib/pdfUtils.ts` 新增 `embedCustomBlocks` 函式，在匯出時將自訂區塊繪製到 PDF 頁面
- `ExportButton` / `fillAndExport` 接收 `customBlocks` 參數

**驗收條件**：
- [ ] `CustomTextBlock` 包含：`id`, `type: 'text'`, `page`, `x`, `y`, `width`, `height`, `text`, `fontSize`, `color`
- [ ] `CustomImageBlock` 包含：`id`, `type: 'image'`, `page`, `x`, `y`, `width`, `height`, `imageData` (base64), `imageType` ('png'|'jpeg')
- [ ] `embedCustomBlocks` 能將文字區塊用 `pdfDoc.embedFont` + `page.drawText` 繪製到指定頁面與座標
- [ ] `embedCustomBlocks` 能將圖片區塊用 `pdfDoc.embedPng` / `embedJpg` + `page.drawImage` 嵌入指定頁面與座標
- [ ] 座標轉換：接收 UI 層的任意縮放比例座標，轉換為 PDF 原始座標（需考慮 PDF 預覽縮放）
- [ ] 無自訂區塊時不影響既有匯出行為
- [ ] `npx tsc --noEmit` 無錯誤

**依賴關係**：
- US-002（需 pdfUtils 既有匯出流程）
- US-007（需 ExportButton 與 fillAndExport 整合）

**優先級**：P1
**相關功能**：自訂區塊（規格新增加 §3.5）

#### 驗收說明

**整體結論**：PASS ✅

> 所有驗收條件均已通過。型別定義完整，匯出函式正確嵌入文字與圖片區塊，座標轉換確實處理了 PDF 預覽縮放，無自訂區塊時不影響既有匯出行為，TypeScript 型別檢查無誤。有一項 minor 建議：`embedCustomBlocks` 中文字顏色目前硬編碼為黑色，未使用 `CustomTextBlock.color` 屬性。
