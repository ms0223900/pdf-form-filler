### US-008a：型別擴充 + 素材管理頁面基礎架構

**作為** 使用者
**我想要** 進入 `/materials` 頁面看到分類 tab，並能在各分頁下管理不同類型的素材
**以便** 集中管理所有可重複使用的填表資料

**輸入格式**：
- IndexedDB 中的 `materials` 資料
- 現有 `Material` 型別（`lib/types.ts`）

**輸出格式**：
- 更新 `lib/types.ts` — 新增 `TextMaterialData`、`ImageMaterialData`，`Material.type` 擴充為 `'personal_info' | 'signature' | 'text' | 'image'`
- 更新 `lib/materialStore.ts` — 函式簽章型別同步擴充（`getMaterials`、`addMaterial` 的 type 參數）
- `src/app/materials/page.tsx` — 頁面外殼，含 4 個 tab 切換（個人資料／文字／圖片／簽名）、empty state
- 更新 `src/components/MaterialCard.tsx` — 支援 `'text' | 'image'` 類型的摘要顯示
- 更新 `src/components/MaterialPanel.tsx` — 新增 `onApplyText`、`onApplyImage` callback，`handleSelect` 補上對應分支
- 更新 `src/app/fill/[id]/page.tsx` — 補上 `handleApplyText`、`handleApplyImage` callback 並傳入 MaterialPanel

**驗收條件**：
- [x] `lib/types.ts` 新增 `TextMaterialData { text: string }` 與 `ImageMaterialData { dataUrl: string; imageType: 'png' | 'jpeg' }`
- [x] `Material.type` 擴充為 `'personal_info' | 'signature' | 'text' | 'image'`
- [x] `materialStore.ts` 中 `getMaterials`、`addMaterial` 的型別參數同步更新
- [x] `/materials` 頁面可開啟，顯示 4 個 tab（無內容時各 tab 顯示 placeholder）
- [x] 無任何素材時顯示 empty state 引導
- [x] MaterialPanel 可處理 text/image 素材的點擊套用
- [x] 點擊文字素材 → 在 PDF 目前頁面建立預填文字的文字區塊
- [x] 點擊圖片素材 → 在 PDF 目前頁面置中新增圖片區塊
- [x] `npx tsc --noEmit` 無錯誤

**依賴關係**：
- US-003（需 materialStore 已存在）
- US-006a（需 MaterialPanel 已整合至填寫頁）

**優先級**：P1
**相關功能**：素材管理（spec §3.3）
