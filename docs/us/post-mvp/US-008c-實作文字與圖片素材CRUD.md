### US-008c：文字 + 圖片素材 CRUD

**作為** 使用者
**我想要** 在 `/materials` 頁面新增文字片段（如公司名稱、統編）與上傳圖片（如印章、Logo），並可刪除
**以便** 在填寫 PDF 時快速套用這類素材

**輸入格式**：
- 使用者輸入的文字（素材名稱 + 文字內容）
- 使用者上傳的圖片檔案（PNG/JPG）
- IndexedDB 中的 `materials` 資料（透過 materialStore CRUD）
- US-008a 完成的頁面框架

**輸出格式**：
- 更新 `src/app/materials/page.tsx` — 「文字」與「圖片」tab 的 CRUD UI

**驗收條件**：
- [ ] 「文字」tab 顯示已儲存文字素材列表（素材名稱 + 文字內容預覽）
- [ ] 點擊「新增」可輸入素材名稱與文字內容
- [ ] 新增後列表即時更新
- [ ] 可刪除文字素材（需確認對話框）
- [ ] 無文字素材時顯示提示引導新增
- [ ] 「圖片」tab 顯示已儲存圖片素材列表（圖片縮圖 + 素材名稱）
- [ ] 點擊「新增」可選擇 PNG/JPG 檔案
- [ ] 選取檔案後即時存入素材庫（轉為 base64 dataUrl）
- [ ] 可刪除圖片素材（需確認對話框）
- [ ] 無圖片素材時顯示提示引導新增
- [ ] `npx tsc --noEmit` 無錯誤

**依賴關係**：
- US-008a（需頁面框架與型別定義已就緒）

**優先級**：P1
**相關功能**：素材管理 — 文字／圖片（spec §3.3）

---
#### 注意事項

- 文字素材與圖片素材**不需編輯功能**（簡單可重加）
- 圖片素材儲存為 base64 dataUrl，與 `CustomImageBlock.imageData` 格式一致
