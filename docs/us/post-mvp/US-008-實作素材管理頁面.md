### US-008：實作素材管理頁面

**作為** 使用者
**我想要** 在 `/materials` 頁面管理我的個人資料、文字片段、圖片與簽名素材
**以便** 增刪改查常用資料，在填表時快速套用

**輸入格式**：
- IndexedDB 中的 `materials` 資料
- 使用者輸入的個人資料（姓名、電話、Email、地址、統編等）
- 使用者輸入的文字片段（公司名稱、統一編號、地址等可重複使用的文字）
- 使用者上傳的圖片（印章、Logo、圖章等 PNG/JPG 圖檔）
- 使用者繪製的簽名

**輸出格式**：
- `src/app/materials/page.tsx` — 素材管理頁面
- `src/components/PersonalInfoForm.tsx` — 編輯個人資料表單
- 更新 `lib/types.ts` — `Material` 型別擴充 `'text' | 'image'` 類型

**驗收條件**：
- [ ] 頁面分頁顯示各類素材列表（Tab 切換：個人資料／文字／圖片／簽名）
- [ ] 無任何素材時顯示提示引導使用者新增
- [ ] 個人資料列表顯示素材名稱、內容摘要、建立時間
- [ ] 文字素材列表顯示素材名稱、文字預覽
- [ ] 圖片素材列表顯示圖片縮圖
- [ ] 簽名列表顯示簽名預覽縮圖
- [ ] 可新增個人資料素材（表單：名稱、電話、Email、地址、統編）
- [ ] 可新增文字素材（表單：素材名稱、文字內容）
- [ ] 可新增圖片素材（檔案上傳 PNG/JPG，存入 base64 dataUrl）
- [ ] 可編輯既有個人資料素材
- [ ] 可刪除素材（需確認對話框）
- [ ] 可從素材管理頁開啟簽名繪製器並儲存
- [ ] `npx tsc --noEmit` 無錯誤

**依賴關係**：
- US-003（需 materialStore）
- US-006a（需 MaterialPanel 已整合至填寫頁，以便套用新增的素材類型）

**優先級**：P1
**相關功能**：素材管理（spec §3.3）

---
#### 型別擴充

`lib/types.ts` 需新增：

```typescript
export interface TextMaterialData {
  text: string;
}

export interface ImageMaterialData {
  dataUrl: string;
  imageType: 'png' | 'jpeg';
}

// Material.type 擴充為：
// type: 'personal_info' | 'signature' | 'text' | 'image'
```

素材套用行為：
- **文字素材** → MaterialPanel 點擊時在目前 PDF 頁面建立一個預填文字的文字區塊
- **圖片素材** → MaterialPanel 點擊時在目前 PDF 頁面置中新增圖片區塊（與簽名行為一致）
